웹 브라우저란? 서버에서 받아온 HTML CSS JS를 실행시켜주는 프로그램 근데 브라우저가 자바스크립트를 실행하는데 일련의 과정이 있다. 브라우저는 C++ 이라는 언어로 코드가 짜여져있음. 이 브라우저는 실행해야할 자바스크립트 코드를 발견하면 C++ 언어로 만들어둔 stack에 넣어 돌림. stack은 다 집어넣고 맨 윗줄부터 하나하나 실행시키는 공간. 하지만 가끔가다가 특별한 코드를 실행해야할 경우가 있음. (서버로의 ajax 요청, 이벤트리스너, setTimeout 이런 코드) 이런 코드는 처리하기까지 시간이 오래걸림, ajax 요청은 서버에서 응답을 받기까지 시간이 오래걸리고 버튼 이벤트리스너는 사용자가 버튼을 누르기까지 시간이 오래걸림. 그래서 그런건 Stack에 쌓아서 실행하지 않고 잠깐 보류해놨다가 완료가 되는 시점에 Stack으로 보냄. 근데 Stack은 항상 바빠서 Stack이 텅 빈 시점에 집어넣게 되어있음. ajax 요청, 이벤트리스너, setTimeout 이런 코드가 실행준비가 되면 Queue라는 곳에 집어넣고 Queue에 있던 코드는 Stack으로 옮겨서 실행해줌. Stack에 넣어서 실행하는데 Stack이 비어있을 때만 차례로 집어넣어서 실행해줌. (참고로 Queue는 들어온 순서대로 차례차례 Stack으로 옮겨줌.) Stack을 바쁘게 만들면 ajax 요청, 이벤트리스너, setTimeout 이런 코드 실행이 불가능함. 10초 걸리는 코드가 있으면 Stack이 10초동안 비지 않기 때문에 10초 동안 ajax 요청, 이벤트리스너, setTimeout 이런 코드는 실행이 불가능함. 그래서 Stack을 생각하며 코드 짜야함. for (let i = 0; i < 1e10; i++) { i++; } 이런 작업을 해야한다면, 1. setTimeout을 이용 setTimeout()을 이용해서 0초마다 0~1억 반복, 1억~2억 반복, 2억~3억 반복... 이렇게 코드를 실행하면 보다 쾌적하게 작업을 실행할 수 있음. 0초마다 Queue로 보내기 때문에 그 사이사이에 사용자의 이벤트리스너 이런 코드를 실행할 수 있음. (setTimeout 타이머를 0초로 설정해도 실은 4ms로 동작함.(설정가능한 최소시간이 4ms)) 2. Web worker를 이용 다른 자바스크립트 파일을 이용해서 그 파일에서 힘든 연산을 시키고 그게 완료가 되면 값을 가져오라고 명령하면 됨. => Worker라는 클래스를 사용 /// (js 파일) var myWorker = new Worker('worker.js'); w.onmessage = function(e){ console.log(e.data) // 1 출력 }; (worker.js) var i = 0; postMessage(i + 1); //postMessage라는 특별한 함수 /// worker.js에서 작업이 완료될 시 postMessage() 이렇게 실행하면 다른 파일로 완료된 결과값을 전달해줄 수 있음. 이러면 Stack이 바빠지는 것을 방지함
